### 🔭 요즘 관심사

**Elixir/OTP** — 프로세스별 독립 힙 + 마이크로초 단위 GC가 주는 p99 레이턴시 안정성에 꽂혔다. BEAM VM의 Actor 모델, GenServer 상태 관리, Supervisor Tree 장애 격리를 직접 예약 플랫폼에 적용 중. ETS로 Redis 없이 인메모리 캐시 구현하는 구조까지 파고 있고, "Go 쓸 일에 Elixir 쓰면 된다"는 결론에 도달함.

**Rust WASM** — SoundScape 프로젝트에서 3D 공간 오디오 엔진을 Rust로 직접 만들고 있다. Web Audio API 위에 wasm-bindgen으로 브라우저 바인딩하고, Tauri에서는 cpal로 네이티브 I/O 처리. 크로스 플랫폼 비즈니스 로직을 WASM으로 한 번 작성해서 웹/모바일/데스크탑에 뿌리는 아키텍처에 관심이 많다. TripHorizon 프로젝트에서 자체 지도 페이지를 직접 만들면서 최적화 방향에 사용 중.

**Post-Quantum Cryptography** — Zero trust network 와 마찬가지로, 보안에서 개발자가 앞으로 챙겨야 하는 스택 중 하나로 생각되어 공부 중.

**Godot 4 Game Dev** — 쿼터뷰 픽셀아트 인디 게임 개발 중. 3D 배경 + 2D 캐릭터 하이브리드 방식으로 Cerbere Games 화풍을 참고하고 있다. GDScript로 CharacterBody3D 이동, AnimationPlayer, 카메라 리그까지 구현했고, 사운드는 sfxr + ChipTone으로 직접 제작. Live2D 컷씬 연동도 검토 중.

**Relay (GraphQL)** — 예약 플랫폼 프론트엔드에서 Relay를 도입. Fragment 기반 데이터 패칭과 컴파일 타임 쿼리 검증이 대규모 컴포넌트 트리에서 주는 이점에 주목하고 있다.

**Bun Runtime (Hono, Elysia)** — Node.js 대안으로 Bun을 적극 도입 중. Elysia의 엔드투엔드 타입 추론과 Express 대비 10~20배 성능에 매력을 느껴서 여행 플랫폼의 Auth/Notification 서비스에 적용했다. "Go 쓸 일에 Elixir, Node 쓸 일에 Bun"이라는 판단.

**Nx Monorepo** — 기존에 turborepo로 관리 중인 모든 프로젝트를 Nx로 관리 중. apps/packages/modules 구조로 웹(Astro), 데스크탑(Tauri), 모바일(React Native), WASM(Rust)을 하나의 저장소에서 운영. claude code를 이용한 풀스택 개발에 있어 모든 코드를 공유하는 모노레포가 적합하다 생각하여 더 중요하다 생각 중.

**Micro-Frontend** — Single-SPA로 앱 라이프사이클 오케스트레이션하고, Rspack Module Federation, Vite Module Ferderation 으로 런타임 모듈 공유하는 구조를 직접 설계·구현 중. MFE 사용에 있어서 아일랜드 패턴, dynamic routing 등의 여러가지 사용법을 체험 중. Nextjs, Astro와 같은 SSR 플랫폼과도 비교 및 같이 사용하며 SEO도 좋게하고 mfe 도 쓸 수 있는 방안 탐색 중.
